## [개념 정리] Kafka 서버 구동의 필요성
- **Kafka 서버 (Docker)**: 실제 메시지 저장 및 전달 역할을 하는 인프라. 애플리케이션 실행 전 반드시 기동되어야 함.
- **Spring Kafka (Code)**: 애플리케이션 내에서 Kafka 서버와 통신하기 위한 라이브러리.
- **결론**: 엔진(Kafka)은 도커로 띄우고, 조작(Spring)은 우리 코드로 수행함.

## [개념 정리] Kafka Bootstrap Server 주소의 의미
- **Bootstrap Servers (`localhost:9092`)**: 애플리케이션 내장 실행이 아니라, Docker로 띄운 외부 Kafka 서버에 접속하기 위한 '주소'임.
- **포트 포워딩**: Docker Compose의 포트 설정을 통해 로컬 호스트의 9092 포트와 컨테이너 내부의 9092 포트가 연결됨.
- **결론**: 애플리케이션은 로컬에서 실행되더라도, 네트워크를 통해 외부에 있는 Kafka 브로커와 통신함.

## [완료] Spring Kafka 애플리케이션 적용 (2단계)
- **의존성 및 설정 완료**: 모든 모듈에 `spring-kafka` 의존성 및 `application.yaml` 설정 추가 완료.
- **코드 리팩토링 완료**:
  - `OrderService`: Kafka 발행자로 전환.
  - `StockEventListener`, `PointEventListener`: Kafka 구독자 및 결과 발행자로 전환.
  - `OrderEventListener`: 외부 결과 메시지 구독자로 전환.
- **결과**: 물리적으로 분리된 서비스들이 Kafka를 통해 비동기 Saga 패턴을 수행할 수 있는 구조 완성.
