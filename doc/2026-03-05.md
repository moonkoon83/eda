- **분석**: `@EventListener` 애너테이션과 Spring의 `ApplicationEventPublisher` 동작 방식 설명.

## 기본 프로필 설정 변경 (완료)
- **작업**: 명시적 프로필 지정이 없는 경우 기본적으로 `local` 프로필을 활성화하도록 설정 변경.
- **방식**: `application.yaml`에 `spring.profiles.active: local` 추가 완료.
- **결과**: 별도 옵션 없이 실행 시 `local` 설정(MySQL)을 기본으로 사용함.

## 로컬 환경 데이터베이스 설정 확인 (해결)
- **원인**: 애플리케이션 실행 시 프로필(`spring.profiles.active`)을 지정하지 않음.
- **분석 결과**:
    - 프로필 미지정 시 `application.yaml`만 로드됨.
    - `application.yaml`에 데이터소스 설정이 없으므로, Spring Boot Auto-configuration에 의해 클래스패스의 H2로 자동 연결됨.
- **해결 방법**: `application.yaml`에 기본 프로필로 `local`을 지정함.
